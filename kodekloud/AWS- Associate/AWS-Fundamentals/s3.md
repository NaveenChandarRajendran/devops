- It is the object based storage.
- It can be access by the ways like **AWS Console,AWS CLI,SDK,Rest API**
- It cannot be NFS or Server Side Block [storage. It](http://storage.It) is standalone object-based storage.
- We can create as many s3 bucket for our preference like one logging, videos etc

**Objects**

- What ever we upload in the s3 it will stored as a object
- The object has the properties of key(file name),value(file data),other info like meta data etc.

**S3 File Structure**

- S3 has the flat file structure not the directory wise structure.
- If you upload file like music/song1, music/song2, music/song3 means. The amzon send the response like it all put inside the same directorory like music folder, but in behind it just a direct files which is uploaded

**Availability**

- If we upload a file means the S3 will be stored in not only in one [place.It](http://place.It) will be stored in multiple avaiability zone, so even the service or avaiability zone is failed ,we can get the data from the other avaibility zones.

**S3 Bucket Names**

- The naming of the S3 bucket should be unique with respect to globally.

**S3 Restrictions**

- S3 handle unlimited number of objects
- Maximun size of a single file is 5TB
- An Aws account supports **100 buckets** by default,but this number can be increased to 1000 by requesting an **service limit increase**
