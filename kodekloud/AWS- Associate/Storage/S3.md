S3

Object based  storage from Amazon, it has high scalability, data availabity, security and performance.

- Use cases include static websites, media files,logs and traces  and much more.
- This object based is flat file structure so we cant have directory wise storing feature
- They Object has properties of key - fileaname, value - acutal value and other some properties like metadata etc.
- Buckets is called in terms of object containers
- Bucket names must be unique because we can access the object with the bucket name based url.
- S3 can handle unlimited number of objects
- Maximum size of the individual object is 5 TB.
- Multi-part uploads allows you to break up an object into parts before uploading.

**S3- Versioning**

- This is the feature that allows you to preserve, retrive and restore every version of the object stored in your bucket.
- Versioning is in disabled state by default we need to enable it manually.
- Versioning can be enabled at the bucket level not on individual objects.
- The bucket version has three types
    - Unversioned - (Default)
    - Enabled
    - Suspended
- Once the bucket is enabled on a bucket,it cannot move back to an unversioned state and can only be in the suspended state.
- When its on suspend state, previous versions remain, but no new versions will be created.
- Users are charged for each versions of an object.(Total size of the versions is charged)
- MFA-(Multi factor authentication) can be configured to secure the versioning state of a bucket.

**ACLs and Resource policy**

- Determines who can access to the bucker and what operations they can perform.
- Within a policy you have json format which has the properties of Principal, Resource,Effect and Action.
    - Principle determines who the policy should apply to.
    - Resources determines what AWS resources the policy should apply to
    - Action determines what the principal is allowed to perform on the resources
    - Effect either allows or denies an action.
- Bucket policies work alongside IAM policies.
- ACLs are legacy access control method that predate IAM.

**Pre Signed URL**

- This is url which will give access to S3 for the limited time.
- When user clicks on a pre-signed URL,they are performing a request to AWS API with the identity of the user that created the pre-signed URL
- If the user that creates a pre-signed URL cannot access an object,then the user clicks the URL will also not be able to access the object
